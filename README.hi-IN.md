## वेब यूआई

**`PAGES_URL`** — अपने फोर्क के लिए वेब यूआई बेस यूआरएल को कॉन्फ़िगर करने के लिए इसे रिपॉजिटरी वेरिएबल के रूप में सेट करें।

एक साफ, डार्क-थीम वाले इंटरफ़ेस में सभी ऐतिहासिक डाइजेस्ट ब्राउज़ करें — लॉगिन की आवश्यकता नहीं है। रिपोर्ट इस रिपॉजिटरी में मौजूद मार्कडाउन फ़ाइलों से GitHub Pages के माध्यम से प्रदर्शित की जाती हैं।

वेब यूआई `packages/web/` में स्थित एक Vite + TypeScript SPA है। यह रनटाइम पर `manifest.json` और व्यक्तिगत डाइजेस्ट फ़ाइलें प्राप्त करता है — नए दैनिक डाइजेस्ट बिना किसी यूआई पुनर्निर्माण के दिखाई देते हैं।


```bash
bun run dev:web    # start Vite dev server for local UI development
bun run build:web  # build the web UI for deployment
```

## RSS फ़ीड

**`PAGES_URL`/feed.xml** — नए डाइजेस्ट स्वचालित रूप से प्राप्त करने के लिए किसी भी RSS रीडर (Feedly, Reeder, NewsBlur, आदि) में सदस्यता लें। फ़ीड URL आपकी `PAGES_URL` सेटिंग से प्राप्त होता है। फ़ीड में सभी प्रकार की नवीनतम 30 रिपोर्ट शामिल हैं, जिन्हें `manifest.json` के साथ प्रतिदिन अपडेट किया जाता है।

## MCP सर्वर

**`https://agents-radar-mcp.duanyytop.workers.dev`**

एक होस्टेड [मॉडल कॉन्टेक्स्ट प्रोटोकॉल](https://modelcontextprotocol.io) सर्वर जो एजेंट्स-रडार डेटा को टूल के रूप में प्रदर्शित करता है। कोई भी MCP-संगत क्लाइंट (Claude Desktop, OpenClaw, आदि) नवीनतम AI इकोसिस्टम रिपोर्ट को सीधे क्वेरी कर सकता है।

**उपलब्ध उपकरण:**

| उपकरण | विवरण |

|------|-------------|

| `list_reports` | उपलब्ध तिथियां और रिपोर्ट प्रकार (पिछले N दिन) सूचीबद्ध करें |

| `get_latest` | किसी दिए गए प्रकार की नवीनतम रिपोर्ट प्राप्त करें |

| `get_report` | तिथि और प्रकार के आधार पर विशिष्ट रिपोर्ट प्राप्त करें |

| `search` | हाल की रिपोर्टों में कीवर्ड खोज |

**क्लाउड डेस्कटॉप सेटअप** — इसे `~/Library/Application Support/Claude/claude_desktop_config.json` में जोड़ें:


```json
{
  "mcpServers": {
    "agents-radar": {
      "url": "https://agents-radar-mcp.duanyytop.workers.dev"
    }
  }
}
```
सेव करने के बाद क्लाउड डेस्कटॉप को रीस्टार्ट करें। फिर आप क्लाउड से ये सवाल पूछ सकते हैं:

- *"एआई सीएलआई टूल्स में नवीनतम क्या है?"* → `get_latest` कमांड का उपयोग करें
- *"इस सप्ताह क्लाउड कोड के उल्लेख खोजें"* → `search` कमांड का उपयोग करें
- *"मुझे 2026-03-05 के लिए एआई ट्रेंडिंग रिपोर्ट दिखाएं"* → `get_report` कमांड का उपयोग करें

**ओपनक्लॉ सेटअप** — निम्न कमांड चलाएँ:


```bash
openclaw mcp add --transport http agents-radar https://agents-radar-mcp.duanyytop.workers.dev
```
या फिर इसे मैन्युअल रूप से `~/.openclaw/openclaw.json` में जोड़ें:

```json
{
  "mcpServers": {
    "agents-radar": {
      "type": "http",
      "url": "https://agents-radar-mcp.duanyytop.workers.dev"
    }
  }
}
```
इसके बाद आप OpenClaw से निम्न प्रकार के प्रश्न पूछ सकते हैं:

- *"AI CLI टूल्स में नवीनतम क्या है?"* → `get_latest` फ़ंक्शन को कॉल करता है
- *"इस सप्ताह क्लाउड कोड के उल्लेखों की खोज करें"* → `search` फ़ंक्शन को कॉल करता है
- *"मुझे 2026-03-05 के लिए AI ट्रेंडिंग रिपोर्ट दिखाएं"* → `get_report` फ़ंक्शन को कॉल करता है

**स्वयं होस्टिंग** — `mcp/` डायरेक्टरी से अपना स्वयं का इंस्टेंस डिप्लॉय करें:


```bash
cd mcp
pnpm install
wrangler deploy
```

## विशेषताएं

- सभी ट्रैक किए गए रिपॉजिटरी में पिछले 24 घंटों में अपडेट किए गए इश्यू, पुल रिक्वेस्ट और रिलीज़ को फ़ेच करता है।
- ट्रेंडिंग क्लाउड कोड स्किल्स को ट्रैक करता है — समुदाय की सहभागिता के आधार पर क्रमबद्ध, न कि नवीनता के आधार पर।
- प्रत्येक CLI रिपॉजिटरी के लिए प्रति-टूल सारांश और क्रॉस-टूल तुलनात्मक विश्लेषण उत्पन्न करता है।
- एक विस्तृत ओपनक्लॉ प्रोजेक्ट रिपोर्ट और 11 समकक्ष परियोजनाओं के साथ क्रॉस-इकोसिस्टम तुलना उत्पन्न करता है।
- साइटमैप के माध्यम से आधिकारिक एंथ्रोपिक और ओपनएआई वेब सामग्री को स्क्रैप करता है; नए लेखों का क्रमिक रूप से पता लगाता है।
- GitHub ट्रेंडिंग की दैनिक निगरानी करता है + 6 AI विषय टैग खोजता है; आयाम के आधार पर रिपॉजिटरी को वर्गीकृत करता है और ट्रेंड सिग्नल निकालता है।
- हैकर न्यूज़ से शीर्ष 30 AI कहानियों को फ़ेच करता है (पिछले 24 घंटे, अंकों के आधार पर रैंक की गई); समुदाय की भावना रिपोर्ट उत्पन्न करता है।
- प्रत्येक रिपोर्ट प्रकार के लिए GitHub इश्यू प्रकाशित करता है; मार्कडाउन फ़ाइलों को `assets/digests/YYYY-MM-DD/` में कमिट करता है।
- GitHub Actions के माध्यम से दैनिक शेड्यूल पर चलता है। मैन्युअल ट्रिगरिंग का समर्थन करता है
- सभी ट्रैक किए गए रिपॉजिटरी को `config.yml` के माध्यम से कॉन्फ़िगर किया जा सकता है — किसी कोड परिवर्तन की आवश्यकता नहीं है
- `locales/*.json` के माध्यम से केंद्रीकृत लोकेल सिस्टम — `packages/core/src/locales/t.ts` में `t()` कैटलॉग के साथ 21 समर्थित भाषाएँ

## सेटअप

1. इस रिपॉजिटरी को **फोर्क** करें
2. अपने LLM प्रदाता कुंजी को रिपॉजिटरी सीक्रेट के रूप में जोड़ें (उदाहरण के लिए `ANTHROPIC_API_KEY`)
3. **एक्शन** टैब में वर्कफ़्लो को सक्षम करें — इसके बाद यह स्वचालित रूप से चलता है

पूरी गाइड के लिए — सीक्रेट संदर्भ, LLM प्रदाता, `config.yml` अनुकूलन, स्थानीय विकास, बहु-भाषा कॉन्फ़िगरेशन और शेड्यूल — [docs/setup.md](./docs/setup.md) देखें।


## आउटपुट प्रारूप

प्रत्येक सक्षम भाषा के लिए प्रति रिपोर्ट प्रकार एक फ़ाइल, जो `assets/digests/YYYY-MM-DD/` फ़ोल्डर में लिखी जाती है। ऐतिहासिक डाइजेस्ट [`assets/digests/`](./assets/digests/) में उपलब्ध हैं। पूर्ण फ़ाइल सूची, प्रति-रिपोर्ट संरचना और GitHub इश्यू लेबल संदर्भ के लिए [docs/output-format.md](./docs/output-format.md) देखें।

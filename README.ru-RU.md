Рабочий процесс GitHub Actions, запускающийся каждое утро в 08:00 по центральному стандартному времени. Он агрегирует сигналы экосистемы ИИ из 10 источников данных, а затем публикует ежедневные сводки (на всех настроенных языках) в виде задач GitHub Issues и зафиксированных файлов Markdown. Еженедельные и ежемесячные сводные отчеты также генерируются автоматически.

## Веб-интерфейс

**`PAGES_URL`** — Установите это значение в качестве переменной репозитория, чтобы настроить базовый URL веб-интерфейса для вашего форка.

Просматривайте все исторические дайджесты в чистом интерфейсе с темной темой — вход в систему не требуется. Отчеты формируются из файлов Markdown в этом репозитории через GitHub Pages.

Веб-интерфейс представляет собой одностраничное приложение (SPA) на Vite и TypeScript, расположенное в `packages/web/`. Он загружает `manifest.json` и отдельные файлы дайджестов во время выполнения — новые ежедневные дайджесты появляются без перестройки пользовательского интерфейса.

```bash
bun run dev:web    # start Vite dev server for local UI development
bun run build:web  # build the web UI for deployment
```

## RSS-лента

**`PAGES_URL`/feed.xml** — Подпишитесь в любом RSS-ридере (Feedly, Reeder, NewsBlur и т. д.), чтобы автоматически получать новые дайджесты. URL ленты определяется на основе вашей настройки `PAGES_URL`. Лента включает 30 последних отчетов по всем типам отчетов, обновляемых ежедневно вместе с `manifest.json`.

## Сервер MCP

**`https://agents-radar-mcp.duanyytop.workers.dev`**

Размещенный сервер [протокола контекста модели](https://modelcontextprotocol.io), предоставляющий данные agents-radar в качестве инструментов. Любой MCP-совместимый клиент (Claude Desktop, OpenClaw и т. д.) может напрямую запрашивать последние отчеты экосистемы ИИ.

**Доступные инструменты:**

| Инструмент | Описание |

|------|-------------|

| `list_reports` | Список доступных дат и типов отчетов (за последние N дней) |

| `get_latest` | Получить самый последний отчет заданного типа |

| `get_report` | Получить конкретный отчет по дате и типу |

| `search` | Поиск по ключевым словам в последних отчетах |

**Настройка Claude Desktop** — добавьте в `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "agents-radar": {
      "url": "https://agents-radar-mcp.duanyytop.workers.dev"
    }
  }
}
```
После сохранения перезапустите Claude Desktop. Затем вы сможете задавать Claude такие вопросы, как:

- *«Что нового в инструментах командной строки для ИИ?»* → вызывается `get_latest`
- *«Найдите упоминания Claude Code на этой неделе»* → вызывается `search`
- *«Покажите мне отчет о тенденциях в области ИИ за 05.03.2026»* → вызывается `get_report`

**Настройка OpenClaw** — выполните следующую команду:


```bash
openclaw mcp add --transport http agents-radar https://agents-radar-mcp.duanyytop.workers.dev
```
Или добавьте его вручную в `~/.openclaw/openclaw.json`:

```json
{
  "mcpServers": {
    "agents-radar": {
      "type": "http",
      "url": "https://agents-radar-mcp.duanyytop.workers.dev"
    }
  }
}
```
Затем вы можете задавать OpenClaw такие вопросы, как:

- *«Что нового в инструментах командной строки для ИИ?»* → вызывается `get_latest`
- *«Найдите упоминания Клода Кода на этой неделе»* → вызывается `search`
- *«Покажите мне отчет о тенденциях в области ИИ за 05.03.2026»* → вызывается `get_report`

**Самостоятельное размещение** — разверните свой собственный экземпляр из каталога `mcp/`:


```bash
cd mcp
pnpm install
wrangler deploy
```
## Отслеживаемые источники

10 источников данных из репозиториев GitHub, новостей, исследований и лент сообщества. Полный список со ссылками на репозитории и подробной информацией по каждому источнику см. в [docs/sources.md](./docs/sources.md).

**Краткое описание:** 9 репозиториев AI CLI + OpenClaw + 12 агентов-партнеров (API GitHub) · Claude Code Skills · GitHub Trending (HTML + Search API) · Hacker News · ArXiv · Hugging Face · Product Hunt · Dev.to · Lobste.rs · Карты сайтов Anthropic + OpenAI.

## Особенности

- Получает информацию о проблемах, запросах на слияние и релизах, обновленных за последние 24 часа во всех отслеживаемых репозиториях
- Отслеживает популярные навыки программирования Claude Code Skills — сортировка по активности сообщества, а не по новизне
- Генерирует сводку по каждому инструменту для каждого репозитория CLI и сравнительный анализ между инструментами
- Генерирует подробный отчет по проекту OpenClaw, а также сравнение с 11 аналогичными проектами в рамках всей экосистемы
- Собирает официальный веб-контент Anthropic и OpenAI через карты сайта; постепенно обнаруживает новые статьи
- Ежедневно отслеживает тренды GitHub + выполняет поиск по 6 тегам тем ИИ; классифицирует репозитории по параметрам и извлекает сигналы трендов
- Получает 30 лучших новостей об ИИ с Hacker News (за последние 24 часа, ранжированных по баллам); генерирует отчет об отношении сообщества
- Публикует проблемы GitHub для каждого типа отчета; Создает фиксацию файлов Markdown в `assets/digests/YYYY-MM-DD/`
- Запускается по ежедневному расписанию через GitHub Actions; Поддерживается ручное управление
- Все отслеживаемые репозитории настраиваются через `config.yml` — никаких изменений в коде не требуется
- Централизованная система локализации через `locales/*.json` — поддерживается 21 язык с каталогом `t()` в `packages/core/src/locales/t.ts`

## Настройка

1. **Создайте форк** этого репозитория
2. Добавьте ключ поставщика LLM в качестве секрета репозитория (например, `ANTHROPIC_API_KEY`)
3. Включите рабочий процесс на вкладке **Действия** — с этого момента он будет запускаться автоматически

Полное руководство — справочник секретов, поставщики LLM, настройка `config.yml`, локальная разработка, многоязычная конфигурация и расписание — см. [docs/setup.md](./docs/setup.md).

## Формат вывода

Один файл на каждый тип отчета для каждого включенного языка, записывается в `assets/digests/YYYY-MM-DD/`. Исторические дайджесты находятся в [`assets/digests/`](./assets/digests/). Полный список файлов, структура для каждого отчета и ссылка на метку задачи GitHub см. в [docs/output-format.md](./docs/output-format.md).
